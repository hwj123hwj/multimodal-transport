# 匹配溯源动画 - 设计文档

## 目标

为老师/答辩展示：**Gale-Shapley 稳定匹配算法是如何一步步将货物匹配到路线的**

## 页面结构

### 布局（三栏式）

```
┌──────────────────────────────────────────────────────────┐
│  匹配溯源动画演示                              [货物选择▼] │
├──────────────────────────────────────────────────────────┤

│   货物侧          动画区域（连线动画）         路线侧      │
│                                                          │
│  ┌──────┐                              ┌──────┐         │
│  │货物1  │  ─ ─ ─ ─ 表白 ─ ─ ─ ─ ─ →  │路线#7 │         │
│  │货物2  │                              │路线#10│         │
│  │货物3  │                              │路线#4 │         │
│  │...    │                              │路线#1 │         │

│                                                          │
```

## 动画流程

### 数据准备（后端提供）

需要新增 API：`GET /api/matching/trace-steps`

返回 Gale-Shapley 算法的每一步操作：

```json
{
  "steps": [
    {
      "step": 1,
      "type": "propose",
      "shipment_id": 1,
      "route_id": 10,
      "description": "货物1 向路线#10（长江经济带）表白，匹配成本：¥305,858"
    },
    {
      "step": 2,
      "type": "accept",
      "shipment_id": 1,
      "route_id": 10,
      "description": "路线#10 当前空闲，接受货物1！"
    },
    {
      "step": 3,
      "type": "propose",
      "shipment_id": 2,
      "route_id": 10,
      "description": "货物2 向路线#10（长江经济带）表白，匹配成本：¥298,432"
    },
    {
      "step": 4,
      "type": "accept",
      "shipment_id": 2,
      "route_id": 10,
      "description": "路线#10 还有剩余容量，接受货物2！"
    },
    {
      "step": 5,
      "type": "reject",
      "shipment_id": 6,
      "route_id": 7,
      "description": "货物6 向路线#7 表白 → 路线#7 更喜欢货物8，拒绝！"
    }
  ]
}
```

### 前端动画效果

| 事件 | 动画效果 |
|------|---------|
| 货物表白 | 货物节点高亮 + 虚线动画飞向路线 |
| 路线接受 | 路线节点变绿 + 连线变实线（绿色） |
| 路线拒绝 | 路线节点闪红 + 连线消失 + 货物变灰 |
| 已有匹配被替换 | 旧货物连线变红断开 + 新货物连线变绿 |

### 控制按钮

- **上一步/下一步**：手动控制
- **自动播放**：每步间隔 1.5s，可调速
- **重置**：回到初始状态
- **只看某货物**：筛选只显示某个货物的匹配过程

## 路线入口

侧边栏「匹配溯源」菜单 → 进入动画页面

页面顶部有货物选择器，选择某个货物后：
1. 先播放全局动画（所有货物的匹配过程）
2. 然后高亮该货物的匹配路径

## 技术实现

### 后端

在 `app/routes/trace.py` 新增：
- `GET /api/matching/trace-steps?scene_id=xxx`
- 调用 matching_service 重放 Gale-Shapley 算法，记录每一步操作

### 前端

新建 `frontend/src/pages/Trace/MatchingAnimation.jsx`：
- 使用 CSS animation + React state 控制动画
- 左右两栏用固定定位，中间连线用 SVG 或 CSS border
- 步骤说明区域用 antd Card

### 路由

- `/trace` → 匹配动画页面（侧边栏入口）
- `/trace/:shipmentId` → 某货物的溯源详情页（表格页）

## 优先级

1. ⭐ 先做简化版：只展示**单个货物**的匹配过程（表白→接受/拒绝）
2. 后续优化：展示全局匹配过程
3. 后续优化：加入速度控制、回放等功能

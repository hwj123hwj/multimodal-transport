# 多式联运物流模型体系分析总结

## 1. 模型架构概览

多式联运物流系统包含四个核心数据模型，形成完整的业务闭环：

### 1.1 模型层次结构
```
Network（网络层）
    ↓
Route（路线层） ← → Shipment（货物层）
    ↓
StableMatching（匹配层）
```

### 1.2 模型关联关系

1. **Network ↔ Route**：网络提供拓扑结构，路线基于网络节点
2. **Route ↔ Shipment**：路线提供运输能力，货物需要路线服务
3. **Shipment ↔ StableMatching**：货物参与匹配，匹配结果影响货物状态
4. **Route ↔ StableMatching**：路线参与匹配，匹配结果影响路线负载

## 3. 核心业务能力分析

### 3.1 网络拓扑分析（Network）

**核心功能**：
- 网络结构建模（节点、站点、位置）
- 拓扑分析（最短路径、节点重要性）
- 网络统计（密度、聚类系数、路径长度）

**业务价值**：
- 基础设施规划决策支持
- 关键节点识别和保护
- 网络鲁棒性评估

### 3.2 货物需求管理（Shipment）

**核心功能**：
- 运输需求建模（TEU、起始目标、时间价值）
- 优先级自动分级（4级优先级体系）
- 状态生命周期管理（5种状态转换）

**业务价值**：
- 差异化服务提供基础
- 资源分配优先级指导
- 客户需求价值量化

### 3.3 路线资源管理（Route）

**核心功能**：
- 运输能力建模（容量、成本、时间）
- 动态负载管理（实时容量监控）
- 效率评估优化（综合效率评分）

**业务价值**：
- 运输资源优化配置
- 成本控制和时间优化
- 容量利用率最大化

### 3.4 智能匹配服务（StableMatching）

**核心功能**：
- 稳定匹配算法实现
- 匹配效果评估（匹配率、重启次数等）
- 系统性能监控（CPU时间、迭代次数）

**业务价值**：
- 供需匹配优化
- 系统效率量化评估
- 算法性能持续改进

## 4. 数据流转分析

### 4.1 数据输入流程

```
CSV文件 → 模型解析 → 数据验证 → 业务对象
    ↓
Network: network.csv
Shipment: shipments.csv（预期）
Route: routes.csv（预期）
StableMatching: stable_matching.csv
```

### 4.2 业务处理流程

```
货物需求 → 路线筛选 → 容量检查 → 稳定匹配 → 状态更新
   ↓           ↓           ↓           ↓           ↓
Shipment   Route    Route    StableMatching  Shipment+Route
创建       查询     验证     算法执行       状态更新
```

### 4.3 数据输出流程

```
业务对象 → 字典转换 → API响应 → 前端展示
   ↓
to_dict()方法提供标准化数据格式
支持JSON序列化和前端消费
```

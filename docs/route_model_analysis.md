# Route模型分析文档

## 1. 模型概述

Route模型用于表示多式联运路线，包含路线ID、节点序列、运输成本、旅行时间、容量等关键信息，以及路线状态管理和效率评估功能。

## 2. 字段分类

### 2.1 CSV原有字段（基于解析逻辑推断）

由于routes.csv文件不存在，基于from_csv_row方法的解析逻辑推断：

1. **route_id** - 路线ID（第1列）
2. **nodes** - 节点列表（第4-12列，共9个字段）
3. **costs** - 运输成本列表（第13-21列，共9个字段）
4. **travel_times** - 旅行时间列表（第22-30列，共9个字段）
5. **capacity** - 路线容量（第31列）

### 2.2 计算字段（7个）

通过属性方法计算得出的字段：

1. **origin_node** - 起始节点（nodes列表第一个元素）
2. **destination_node** - 目标节点（nodes列表最后一个元素）
3. **total_travel_time** - 总旅行时间（travel_times列表求和）
4. **total_cost** - 总运输成本（costs列表求和）
5. **current_load** - 当前负载（动态计算）
6. **available_capacity** - 可用容量（capacity - current_load）
7. **utilization_rate** - 利用率（current_load / capacity）

### 2.3 状态字段（4个）

1. **status** - 路线状态（AVAILABLE/OCCUPIED/MAINTENANCE/CLOSED）
2. **is_full** - 是否满载（布尔值）
3. **is_available** - 是否可用（布尔值）
4. **efficiency_score** - 效率评分（综合指标）

## 3. 核心方法分析

### 3.1 数据验证与初始化

#### `__post_init__()`
- **功能**：数据验证和后处理
- **逻辑**：
  - 验证节点列表不为空
  - 计算起始和目标节点
  - 验证容量大于0
  - 初始化当前负载为0
  - 设置默认状态为AVAILABLE

### 3.2 容量管理

#### `can_accommodate(demand: int) -> bool`
- **功能**：检查路线是否能容纳指定需求
- **参数**：demand - 货物需求（TEU）
- **逻辑**：比较可用容量与需求

#### `add_load(amount: int) -> bool`
- **功能**：增加负载
- **参数**：amount - 增加的数量
- **逻辑**：
  - 检查是否超载
  - 更新当前负载
  - 更新状态（如满载则设置为OCCUPIED）

#### `remove_load(amount: int) -> bool`
- **功能**：减少负载
- **参数**：amount - 减少的数量
- **逻辑**：
  - 检查是否负载不足
  - 更新当前负载
  - 更新状态（如有空位则设置为AVAILABLE）

### 3.3 效率评估

#### `calculate_efficiency_score() -> float`
- **功能**：计算路线效率评分
- **逻辑**：
  - 基于容量利用率
  - 考虑路线成本效益
  - 综合评估运输效率
- **返回**：0-100的效率评分

### 3.4 CSV数据解析

#### `from_csv_row(cls, row_data: List[str])`
- **功能**：从CSV行数据创建路线实例
- **参数**：row_data - CSV行数据列表
- **逻辑**：
  - 跳过标题行
  - 解析路线ID
  - 解析节点列表（处理-1值）
  - 解析成本和旅行时间
  - 解析容量
  - 计算总成本和总时间

### 3.5 状态管理

#### `update_status(new_status: RouteStatus) -> bool`
- **功能**：更新路线状态
- **参数**：new_status - 新状态
- **逻辑**：验证状态转换的有效性

#### `get_mode_name() -> str`
- **功能**：获取运输方式名称
- **返回**：固定为"多式联运"

#### `get_status_name() -> str`
- **功能**：获取状态中文名称
- **返回**：状态对应的中文名称

## 4. RouteCollection类分析

### 4.1 核心功能

RouteCollection类提供路线集合管理功能：

1. **路线存储**：字典形式存储路线对象
2. **查询过滤**：支持多种条件查询
3. **货物匹配**：为货物匹配合适的路线
4. **统计分析**：提供路线使用统计

### 4.2 主要查询方法

#### `get_available_routes()`
- **功能**：获取可用路线
- **返回**：状态为AVAILABLE的路线列表

#### `get_routes_by_node(node_id: int)`
- **功能**：获取经过指定节点的路线
- **参数**：node_id - 节点ID
- **逻辑**：检查节点是否在路线的nodes列表中

#### `get_routes_by_origin(origin_node: int)`
- **功能**：按起始节点获取路线
- **参数**：origin_node - 起始节点ID

#### `get_routes_by_destination(destination_node: int)`
- **功能**：按目标节点获取路线
- **参数**：destination_node - 目标节点ID

#### `find_routes_for_shipment(origin_node: int, destination_node: int, demand: int)`
- **功能**：为货物匹配合适的路线
- **参数**：
  - origin_node - 货物起始节点
  - destination_node - 货物目标节点
  - demand - 货物需求（TEU）
- **逻辑**：
  - 检查路线连接性
  - 检查容量充足性
  - 按效率评分排序

### 4.3 统计分析

#### `get_statistics()`
- **功能**：获取路线统计信息
- **返回**：包含以下指标的字典：
  - 总路线数、总容量、总负载
  - 平均利用率、可用路线数、满载路线数
  - 运输方式分布、状态分布
  - 总旅行时间、平均旅行时间、平均总成本
  - 效率最高/最低的路线ID

## 5. 业务价值分析

### 5.1 路线资源管理

Route模型帮助管理多式联运路线资源：
- **容量规划**：实时监控路线容量使用情况
- **负载均衡**：优化路线分配，避免某些路线过载
- **资源利用率**：通过效率评分识别低效路线

### 5.2 运输成本控制

通过成本和旅行时间管理：
- **成本分析**：比较不同路线的运输成本
- **时间优化**：选择旅行时间最短的路线
- **综合评估**：平衡成本和时间，选择最优路线

### 5.3 货物匹配优化

RouteCollection的货物匹配功能：
- **智能匹配**：基于起始节点、目标节点和容量需求
- **优先级排序**：按效率评分排序匹配结果
- **容量保证**：确保路线有足够容量容纳货物

### 5.4 运营决策支持

为运营决策提供数据支持：
- **路线维护**：识别需要维护的路线（基于利用率）
- **容量扩展**：识别容量不足的路线
- **网络优化**：基于效率评分优化路线网络

## 6. 扩展性设计

### 6.1 多式联运扩展

- 支持多种运输方式（海运、铁路、公路、航空）
- 不同运输方式的成本和时间模型
- 跨运输方式的路径优化

### 6.2 动态定价

- 基于供需关系的动态定价
- 拥堵时段的价格调整
- 长期合同的优惠定价

### 6.3 预测分析

- 基于历史数据的容量需求预测
- 路线拥堵预测
- 维护需求预测

## 7. 使用建议

### 7.1 数据质量

- 确保CSV文件格式与解析逻辑匹配
- 验证节点序列的连续性
- 定期更新成本和旅行时间数据

### 7.2 性能优化

- 对于大规模路线数据，使用索引加速查询
- 缓存常用的统计结果
- 批量处理路线状态更新

### 7.3 监控告警

- 设置容量利用率告警阈值
- 监控路线状态变化
- 跟踪效率评分异常

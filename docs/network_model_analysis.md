# Network模型分析文档

## 1. 模型概述

Network模型用于表示多式联运物流网络的拓扑结构，包含网络节点、站点信息和位置索引等基础数据，以及网络拓扑分析功能。

## 2. 字段分类

### 2.1 CSV原有字段（3个）

基于network.csv文件分析，直接从CSV解析的字段：

1. **nodes_number** - 网络节点数量（第1行第1列）
2. **sites** - 站点列表（第2行，从第2列开始）
3. **location_indices** - 位置索引列表（第3行，从第2列开始）

### 2.2 计算字段（4个）

通过属性方法计算得出的字段：

1. **node_count** - 节点数量（基于nodes_number）
2. **site_count** - 站点数量（基于sites列表长度）
3. **matching_rate** - 匹配率（预留字段，默认为0.0）
4. **total_capacity** - 总容量（预留字段，默认为0.0）

## 3. 核心方法分析

### 3.1 数据验证与初始化

#### `__post_init__()`
- **功能**：数据验证和后处理
- **逻辑**：
  - 确保sites和location_indices长度一致
  - 创建节点到位置的映射字典
  - 验证节点ID的有效性

### 3.2 CSV数据解析

#### `from_csv_data(csv_data: List[str])`
- **功能**：从CSV数据行创建网络实例
- **参数**：csv_data - CSV文件行列表
- **逻辑**：
  - 解析节点数量
  - 解析站点列表
  - 解析位置索引列表
  - 创建Network实例

#### `from_csv_rows(rows: List[List[str]])`
- **功能**：从CSV行数据创建网络实例
- **参数**：rows - CSV行数据二维数组
- **逻辑**：
  - 跳过标题行
  - 提取网络配置数据
  - 调用from_csv_data创建实例

### 3.3 数据转换

#### `to_dict()`
- **功能**：转换为字典格式
- **返回**：包含所有字段的字典
- **用途**：API响应和数据序列化

### 3.4 查询方法

#### `get_site_by_location(location_index: int)`
- **功能**：通过位置索引获取站点
- **参数**：location_index - 位置索引
- **返回**：站点ID或None

#### `get_location_by_site(site_id: int)`
- **功能**：通过站点获取位置索引
- **参数**：site_id - 站点ID
- **返回**：位置索引或None

## 4. NetworkTopology类分析

### 4.1 核心功能

NetworkTopology类提供网络拓扑分析功能：

1. **邻接矩阵构建**：基于节点连接关系
2. **节点重要性分析**：基于介数中心性
3. **最短路径查找**：使用Dijkstra算法
4. **网络统计信息**：节点度数、聚类系数等

### 4.2 主要方法

#### `build_adjacency_matrix(connections: List[Tuple[int, int]])`
- **功能**：构建邻接矩阵
- **参数**：connections - 节点连接列表
- **逻辑**：创建对称的邻接矩阵表示

#### `calculate_node_importance()`
- **功能**：计算节点重要性（介数中心性）
- **返回**：节点重要性字典
- **算法**：基于最短路径的介数中心性

#### `find_shortest_path(start: int, end: int)`
- **功能**：查找最短路径
- **参数**：start - 起始节点，end - 目标节点
- **返回**：路径节点列表和距离
- **算法**：Dijkstra最短路径算法

#### `get_network_statistics()`
- **功能**：获取网络统计信息
- **返回**：包含多种网络指标的字典
- **指标**：
  - 节点数量、边数量
  - 平均度数、密度
  - 平均路径长度、直径
  - 聚类系数

## 5. 业务价值分析

### 5.1 网络结构理解

Network模型帮助理解多式联运网络的拓扑结构：
- **节点分布**：了解网络中节点的地理分布
- **连接关系**：分析节点间的连接模式
- **网络规模**：掌握网络的复杂程度

### 5.2 网络性能评估

通过NetworkTopology类提供的分析功能：
- **关键节点识别**：找出网络中最重要的节点
- **路径优化**：计算最短路径用于路线规划
- **网络鲁棒性**：评估网络对节点故障的容忍度

### 5.3 决策支持

为网络优化提供数据支持：
- **基础设施投资**：识别需要加强的关键节点
- **容量规划**：基于网络结构优化资源配置
- **风险管控**：识别单点故障风险

## 6. 扩展性设计

### 6.1 动态网络支持

- 支持网络结构的动态更新
- 可添加或删除节点和连接
- 实时更新拓扑分析结果

### 6.2 权重扩展

- 支持边权重（距离、成本、时间等）
- 多权重路径优化
- 权重敏感的重要性分析

### 6.3 多层网络

- 支持多层网络结构（不同运输方式）
- 跨层连接和路径分析
- 多层网络统计指标

## 7. 使用建议

### 7.1 数据质量

- 确保CSV文件格式正确
- 验证节点和位置索引的一致性
- 定期更新网络结构数据

### 7.2 性能优化

- 对于大规模网络，使用稀疏矩阵存储
- 缓存频繁查询的拓扑分析结果
- 并行化节点重要性计算

### 7.3 可视化集成

- 结合网络可视化工具展示拓扑结构
- 突出显示关键节点和路径
- 交互式网络探索界面

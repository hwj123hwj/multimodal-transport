# Shipment模型分析文档

## 1. 模型概述

Shipment模型用于表示货物运输需求，包含货物ID、起始节点、目标节点、需求量（TEU）、时间价值等核心属性，以及货物状态管理和优先级评估功能。

## 2. 字段分类

### 2.1 CSV原有字段（5个）

基于from_csv_row方法的解析逻辑，直接从CSV解析的字段：

1. **shipment_id** - 货物ID（对应"shipment index"字段）
2. **origin_node** - 起始节点（对应"Origin"字段）
3. **destination_node** - 目标节点（对应"destination"字段）
4. **demand** - 需求量（对应"Demand"字段，单位TEU）
5. **time_value** - 时间价值（对应"Time value(CNY/TEU)"字段）
6. **classification** - 分类（对应"Classification"字段，冗余字段，默认为-1）

### 2.2 计算字段（4个）

通过属性方法计算得出的字段：

1. **weight** - 估算重量（demand × 20.0吨/TEU）
2. **volume** - 估算体积（demand × 33.0立方米/TEU）
3. **priority** - 优先级（基于time_value计算，1-4级）
4. **priority_name** - 优先级名称（中文描述）

### 2.3 状态字段（3个）

1. **status** - 货物状态（PENDING/MATCHED/IN_TRANSIT/DELIVERED/CANCELLED）
2. **created_at** - 创建时间
3. **updated_at** - 更新时间

## 3. 核心方法分析

### 3.1 数据验证与初始化

#### `__post_init__()`
- **功能**：数据验证和后处理
- **逻辑**：
  - 初始化创建和更新时间
  - 转换节点数据为整数（处理字符串输入）
  - 验证需求量大于0
  - 验证节点索引不为负数
  - 验证起始节点不等于目标节点
  - 验证时间价值不为负数

### 3.2 物理属性计算

#### `weight`（属性）
- **功能**：估算货物重量
- **逻辑**：demand × 20.0吨/TEU
- **依据**：1TEU标准集装箱约等于20吨

#### `volume`（属性）
- **功能**：估算货物体积
- **逻辑**：demand × 33.0立方米/TEU
- **依据**：1TEU标准集装箱约等于33立方米

### 3.3 优先级评估

#### `priority`（属性）
- **功能**：基于时间价值计算优先级
- **逻辑**：
  - time_value ≥ 2500：4级（最高优先级）
  - time_value ≥ 1700：3级（高优先级）
  - time_value ≥ 500：2级（中等优先级）
  - time_value < 500：1级（低优先级）

#### `get_priority_name()`
- **功能**：获取优先级中文名称
- **返回**：
  - 1级："低优先级"
  - 2级："中等优先级"
  - 3级："高优先级"
  - 4级："最高优先级"

### 3.4 状态管理

#### `update_status(new_status: ShipmentStatus) -> bool`
- **功能**：更新货物状态
- **参数**：new_status - 新状态
- **逻辑**：
  - 定义允许的状态转换规则
  - 验证状态转换的有效性
  - 更新状态和时间戳
- **状态转换规则**：
  - PENDING → MATCHED/CANCELLED
  - MATCHED → IN_TRANSIT/CANCELLED
  - IN_TRANSIT → DELIVERED/CANCELLED
  - DELIVERED/CANCELLED为最终状态

### 3.5 CSV数据解析

#### `from_csv_row(cls, row: Dict[str, str])`
- **功能**：从CSV行数据创建货物实例
- **参数**：row - CSV行数据字典
- **逻辑**：
  - 处理多种字段名称变体
  - 提取货物ID、起始节点、目标节点
  - 解析需求量、时间价值、分类信息
  - 创建Shipment实例

### 3.6 数据转换

#### `to_dict()`
- **功能**：转换为字典格式
- **返回**：包含所有字段的字典
- **包含字段**：
  - 基础信息：shipment_id、origin_node、destination_node
  - 需求信息：demand、time_value、classification
  - 物理属性：weight、volume
  - 优先级信息：priority、priority_name
  - 状态信息：status、created_at、updated_at

## 4. ShipmentCollection类分析

### 4.1 核心功能

ShipmentCollection类提供货物集合管理功能：

1. **货物存储**：字典形式存储货物对象
2. **查询过滤**：支持多种条件查询
3. **状态分组**：按状态、优先级等分组
4. **统计分析**：提供货物需求统计

### 4.2 主要查询方法

#### `get_shipments_by_origin(origin_node: int)`
- **功能**：按起始节点获取货物
- **参数**：origin_node - 起始节点ID

#### `get_shipments_by_destination(destination_node: int)`
- **功能**：按目标节点获取货物
- **参数**：destination_node - 目标节点ID

#### `get_shipments_by_status(status: ShipmentStatus)`
- **功能**：按状态获取货物
- **参数**：status - 货物状态

#### `get_pending_shipments()`
- **功能**：获取待匹配货物
- **返回**：状态为PENDING的货物列表

#### `get_matched_shipments()`
- **功能**：获取已匹配货物
- **返回**：状态为MATCHED的货物列表

#### `get_high_priority_shipments()`
- **功能**：获取高优先级货物（优先级≥3）
- **返回**：优先级为3或4的货物列表

### 4.3 统计分析

#### `get_statistics()`
- **功能**：获取货物统计信息
- **返回**：包含以下指标的字典：
  - 总货物数、总需求量、总重量、总体积
  - 平均时间价值
  - 状态分布（各状态货物数量）
  - 优先级分布（各级别货物数量）
  - 起始节点分布（各起始节点的货物数量）
  - 目标节点分布（各目标节点的货物数量）

## 5. 业务价值分析

### 5.1 货物需求管理

Shipment模型帮助管理货物运输需求：
- **需求量化**：准确表示货物的TEU需求量
- **地理分析**：明确货物的起始和目标位置
- **价值评估**：通过时间价值评估货物的紧急程度

### 5.2 优先级管理

基于时间价值的优先级评估：
- **智能分级**：自动将货物分为4个优先级
- **资源分配**：优先处理高价值货物
- **服务差异化**：为不同优先级提供差异化服务

### 5.3 状态跟踪

完整的货物状态生命周期管理：
- **流程透明**：跟踪货物从待匹配到交付的全过程
- **异常处理**：支持取消等异常状态
- **时效监控**：记录状态转换时间

### 5.4 容量规划

基于物理属性的容量需求分析：
- **重量估算**：基于TEU转换估算总重量
- **体积估算**：基于TEU转换估算总体积
- **资源规划**：为运输工具选择提供依据

### 5.5 网络分析

ShipmentCollection的地理分布分析：
- **流量分析**：分析各节点的货物流入流出
- **热点识别**：识别繁忙的起始-目标节点对
- **平衡优化**：优化网络流量分布

### 7.1 数据质量

- 确保CSV文件字段名称一致性
- 验证节点ID的有效性
- 定期更新时间价值数据

### 7.2 状态管理

- 严格遵循状态转换规则
- 记录状态变更原因
- 设置状态超时告警

### 7.3 性能优化

- 对于大规模货物数据，使用索引加速查询
- 缓存常用的统计结果
